## Context
Emberwalk is a lightweight web research tool (CLI + MCP) that replaces Firecrawl for typical research workflows. It searches, fetches pages with anti-bot defaults and a 3-tier extraction pipeline, and outputs clean markdown. The MCP server exposes three tools for a two-step workflow where the LLM picks which pages to fetch.

## Current State
Fully built, tested, and pushed to DendroLabs/ember-walk (3 commits on main).

Key files:
- emberwalk.py — single-file script, CLI + MCP server (~450 lines)
- install.sh — one-command setup (venv + deps + Playwright chromium)
- benchmark.py — Emberwalk vs Firecrawl comparison on Cisco docs
- requirements.txt — ddgs, requests, trafilatura, beautifulsoup4, lxml, html2text, playwright, playwright-stealth, mcp

MCP config:
- ~/.claude/.mcp.json — "emberwalk" server entry pointing to .venv/bin/python3
- ~/.claude/settings.json — mcp__emberwalk__* permission

Three MCP tools:
- ew_search(query, max_results=20, use_brave=False) — returns JSON snippets, no fetching
- ew_fetch(urls, output_dir="") — fetches specific URLs to markdown files on disk
- ew_research(query, max_results=10) — combined search+fetch (convenience)

Benchmark results (11 Cisco IOS-XR doc pages):
- Emberwalk: 65.7s, 1.35M chars, 11/11 pages
- Firecrawl: 406.1s, 862K chars, 5/11 pages (rest timed out)

## Key Decisions Already Made
- Name: "emberwalk" (riff on Firecrawl — ember = lighter fire, walk = lighter than crawl)
- Single file, not a package
- Two-step MCP workflow: ew_search (snippets) -> LLM picks -> ew_fetch (pages)
- ew_research kept as one-shot convenience tool
- 3-tier extraction: trafilatura -> BeautifulSoup largest container -> full page stripped
- Container extraction picks LARGEST matching selector, not first match
- Project at ~/Documents/emberwalk/ (not misc/)
- Repo: DendroLabs/ember-walk

## Pitfalls to Avoid
- `ddgs` package, NOT `duckduckgo-search` — old name silently returns 0 results
- playwright-stealth v2.x: `Stealth().apply_stealth_sync(page)`, NOT `stealth_sync(page)`
- Moving a venv breaks it (hardcoded shebangs) — recreate at new path
- Trafilatura truncates long technical docs — the 3-tier pipeline handles this
- Container extraction: `<article>` often matches first subsection (~5K chars) while `[role="main"]` matches the full page (~400K chars) — always pick the largest
